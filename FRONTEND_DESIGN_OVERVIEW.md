# 前端设计总览

本文档旨在概述 Sky-Take-Out 项目包含的两个前端应用（后台管理 Web 端与用户端小程序）的技术选型、核心功能及部署模式。

---

## 1. 后台管理系统 (Web 端)

这是提供给商家管理员使用的后台管理界面，通过浏览器访问。

### 1.1. 技术栈

- **核心框架**: **Vue.js**。
- **UI 组件库**: **Element UI** / **Element Plus** (根据打包文件中的 `element-icons` 字体推断)。
- **构建工具**: **Vue CLI** 或类似的现代前端构建工具 (根据打包后的 `app.[hash].js` 和 `chunk-vendors.[hash].js` 文件结构推断)。
- **项目状态**: 当前我们拥有的是编译、打包后的**静态文件**，位于 `nginx-1.20.2/html` 目录中，**没有源代码**。

### 1.2. 主要功能模块 (根据打包文件名推断)

- **登录模块** (`login.js`): 负责用户认证和进入系统。
- **主看板/仪表盘** (`dashboard.js`): 用于展示核心的统计数据和业务概览。
- **表格管理** (`shopTable.js`): 可能用于管理商品、订单、店铺或其他核心数据的表格界面。

### 1.3. 部署方式

- **方式**: 通过 **Nginx** 进行部署。
- **职责**: Nginx 在此扮演双重角色：
    1.  **静态Web服务器**: 直接向用户的浏览器提供网页所需的 HTML, CSS, JS 文件。
    2.  **反向代理**: 将所有 API 请求 (如 `/api/...`) 转发给后端的 Java 服务 (`app` 容器)，从而完美解决跨域问题。

---

## 2. 用户端 (微信小程序)

这是提供给最终消费者在微信中使用的点餐小程序。

### 2.1. 技术栈

- **核心框架**: **uni-app**。这是一个使用 **Vue.js** 语法来开发跨平台应用的框架。
- **UI 组件库**: **uni-ui** (根据 `uni_modules` 目录下的组件推断)。
- **状态管理**: **Vuex** (根据多个页面文件中 `import { mapState } from 'vuex'` 的用法推断)。
- **项目状态**: 我们拥有完整的项目**源代码**，位于 `project-rjwm-weixin-uniapp-develop-wsy` 目录中。

### 2.2. 主要功能模块 (根据 `pages` 目录结构推断)

- **首页 (`pages/index`)**: 商品展示、分类导航、点餐入口。
- **订单页 (`pages/order`)**: 确认订单信息、选择地址、填写备注、提交并支付。
- **“我的”页面 (`pages/my`)**: 用户个人信息展示、登录、历史订单入口、地址管理入口。
- **地址管理 (`pages/address`)**: 新增、编辑、删除用户的收货地址。
- **历史订单 (`pages/historyOrder`)**: 查看过去的订单列表和详情。

### 2.3. 部署方式

- **方式**: 使用 **微信开发者工具** 进行部署。
- **流程**: 在本地开发电脑上，使用微信开发者工具打开此项目源代码，进行调试和预览。开发完成后，通过该工具进行编译，并**上传**代码至腾讯的服务器。经腾讯审核通过后，即可发布给所有微信用户访问。**此过程与我们的 Docker 服务器部署无关**。
